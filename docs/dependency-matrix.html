All modules live under app/src/lib/theme/ and depend only on the M1.3 content stores (published-only reads + getPublicSettings) — never on admin/secret data.

App Auth Hardening (M2.1)

Cross-cutting auth-security primitives every recovery lane (M2.2) and new mutating route consumes, built before those surfaces so they are born compliant (a retrofitted CSRF guard is worse than one built in). Covers CSRF protection, audit logging, a SESSION_SECRET strength floor, and idle-session timeout. Independent security gate = M2.3.

App Layered Recovery Lanes (M2.2)

The single-identity model has no peer-admin recovery, so the layered recovery chain IS the compensating control and is held to the same bar as the primary passkey auth. Holds to the NO-GO gates set for this surface (password-AND-TOTP, break-glass-local-only, no-plaintext-at-rest). Builds on the M2.1 primitives (guardMutation, audit, revokeAllSessions). Single-admin shape (decision D1 locked — a peer-admin recovery lane deliberately NOT built). Independent security gate = M2.3.

Artifact Path Reads / depends on Notes
Password hashing (argon2id) app/src/lib/auth/password.ts Bun.password (runtime) argon2id via the Bun runtime (no native dep in the standalone build). Per-call salt; verifyPassword false on null/garbage, never throws (B1). The fallback lane is password AND TOTP — this is only the password half (NO-GO #3).
TOTP (otplib) app/src/lib/auth/totp.ts otplib (pure-JS Noble crypto) RFC 6238 via otplib (do-not-hand-roll). Window ≤ ±1 step (epochTolerance = one period, T4). verifyTotp returns the matched absolute step (from delta) for one-time-per-step replay rejection (T2). ≥160-bit secret (T1). Epoch-injectable for deterministic tests.
